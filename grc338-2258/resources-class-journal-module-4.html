I learned that shift refresh hard refreshes the page and sometimes Firefox has too much cache so it needs a hard refresh.

I was surprised to see that in a local file rather than on the web.

I recommend “Live Preview” in VS Code to see your edits being made in real time, versus having to keep saving and refreshing on Firefox. Still make sure to double-check on Firefox here and there, but Live Preview definitely saves me a lot of time!

I recommend against it. And I’ll not help you if you don’t have your file open in Firefox.

I was curious how publishing services like Squarespace and Framer deal with alternative text so I looked it up. Pretty interesting, turns out some have options for you to input the information, and some use AI to write an automatic description.

Yes. AI has been quite helpful with that. But as it is often (always?) the case, you have to check its work.

It is important to use SVG files for uploading logos to websites as it is a vector file type rather than a raster image. It is important that when you are creating logos for a website, you should create outlines to preserve the fonts being used.

That’s correct!

Visual Studio Code will help point out errors you may have in your code, and when typing font-__, it will give you options under that menu.

Yes. Then, you know what properties to use. (Don’t make them up.)

Collaboration was super important on this project specifically, in my opinion. I think everyone has gotten to the same level of HTML/CSS knowledge but there were a few gaps in my knowledge that were easily filled in by my labmates, and vice versa. Sometimes it was as simple as looking at my partner’s code and instantly finding a mistake that kept getting past them for 10~ mins. I think a fresh set of eyes is important when there’s so much code involved and I found myself needing help with small little mistakes a lot. It’s mutually beneficially to help out your lab partners as whenever I would help them out, they would return the favor within 10 minutes when I had to ask an equally confusing question.

Exactly. Keep helping each other and checking on each other’s work!

I also have been liking the group work because everyone always has pretty different examples, which help me see the topics in the real world in more places.

I loved the analogy about writing in phone numbers and how even if you are close, as in 9 of the numbers are correct, if one of them is wrong then it won’t call the right number, obviously. Yes. Very good point.

I’m still a little confused on where to put paddings and margins. I put margins in my header, but then I also put them in my h1, h2, etc. It seems to work fine, but I doubt they need to be in both (or they do and I don’t know what I’m talking about).

Your first instinct was correct. You would only need them in the header.

What is the best way to add space between list items while not spacing out the lines of text of individual list items?

If you want to make space between list items, give margin to the list items.

Would you recommend using GitHub to host a portfolio website or a different service?

I do recommend GitHub in general. I host my websites on GitHub Pages. However, you may want to use Squarespace, Wix, or similar service that will give you better content management.
