A declaration in CSS always ends in a semicolon.

For margins/padding, if you don’t define a value for left, it will automatically use the value for right. Same goes for if you don’t define a value for bottom (it will automatically use the value for top).

Correct.

I learned you can do hex code, RGB values, or the typed out name for color.

When listing things in the same row, in the Top Right Bottom Left form specifically, you write the numbers with spaces but NOT with commas. I keep doing that and wondering why it’s not working.

Correct. No commas between the values of the margin or padding properties.

When setting a border above an element, use the CSS line border-top: (pixels) (type) (color);.

And we’ll only use pixels for borders.

Yes. Also, if you have reuploaded your file, but the page still shows the old version, clicking the refresh button on the browser while holding the shift key can do the trick. (Not always, but it tends to work.)
