CSS style: grid layout

Using the files from the last assignments, use CSS Grid to make the page look good on laptop and desktop screens.

Tasks and requirements

  1. Read all the instructions carefully before starting.

  2. Start by reading my comments from the last assignment. Make sure to address every issue on it.

  3. Copy your CSS and JPEG files from last week into the new folder.

  4. Place this code to the bottom of your CSS document. 

    @media only screen and (min-width: 60em) {  



    div.container {  
    max-width: 70em;  
    margin: auto;  
}  


header {  
    padding-top: 4em;;  
}  


figure {  
    display: grid;  
    grid-template-columns: 4fr 1fr;  
    column-gap: 1em;  
}  


main {  
    display: grid;  
    grid-template-columns: 1fr 1fr;  
    column-gap: 3em;  
    padding: 2em 3em;  
}  


figcaption {  
    padding-top: 2em;  
}  


h3 {  
    border-top: #f80 solid 2px;  
    padding-top: 1em;  
}

    

}

  5. This is what your page should look like on the browser.

  6. You want your content centered on the page.

  7. You want your header to take the whole width of the content.

  8. Caption will go on the right of the image.

  9. The Ingredients and Directions sections should take one column each.

  10. The footer must take the whole width of the content.

  11. Check the list of mistakes to avoid.

  12. Keep your CSS organized.

  13. Publish your pages on the URL: username.github.io/grc338/css-grid-layout

Evaluation

I’ll be assessing your ability to create a solution that is visually interesting and appropriate for the content, but you’ll be mostly evaluated on your technical skills using CSS.

Any mistake from the list of mistakes to avoid. will be a 20‑point penalty each. They’re really easy to avoid, though.

Tips

  1. To center your page on the window, give the div.container the properties max-width: XXem; and margin: auto;, replacing XXem with a proper value.

  2. Since only the section elements will be side-by-side, their parent, the main element, will be a grid container.

  3. You’ll need more than one grid container.

  4. Please stop centering type. Your type looks way more professional if you don’t.

  5. Do not change the HTML file for any reason.

  6. Have I asked you to avoid the most common mistakes?

Suggestion

  1. Don’t forget: You want to work offline, saving your files on your computer. Only when you’re happy with the results that you want to upload to GitHub. You may change your mind and upload a new file, but what you don’t want is to work straight on GitHub.

  2. I suggest, however, that you upload the old HTML, CSS, and JPEG files to the new folder in GitHub before you even start changing anything.

  3. Then you check your name. It'll be the last week's layout, of course.

This will prevent you from emailing me the night the assignment is due saying that your file is not working.

Whatever you do, though, you may not edit the HTML file at all.
