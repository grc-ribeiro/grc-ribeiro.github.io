GRC 338: Resources

Be creative on your mistakes

Here’s how you can avoid the ones everybody else makes.

Basics

  1. All your content should be in the <body> of your document.

  2. Close all the tags you opened. (Don’t forget to close <body> and <html>.)

  3. All the pages need to have a <title> in the <head>.

  4. All quotes in the code (not in the content) need to be straight quotes (" not “). It won’t work otherwise. (If you copy your code from another program, it’ll likely come with “smart quote” rather than the straight quotes. And it will break your code.)

  5. In the content, however, use proper apostrophes, quotes, and dashes.

  6. Your homepage file needs to be named index.html or you will get a 403 error.

Type hierarchy

  1. Pieces of text with the same hierarchy should have the same tag. (p, h1, h2, etc.) You may, for example, use many <h2> tags throughout your page.

  2. It never makes sense to have a <p> tag inside a heading tag (<h1>, <h2>, etc).

  3. List items need to live inside lists.

  4. Only list items can be nested inside a list. (Nested inside the list item, though, you may have other elements, such as headings and paragraphs.)

  5. There’s no p2.

  6. There’s no h7.

  7. Avoid line breaks (<br />). Use lists or new paragraphs and style them properly. (That’s not to say that any use of line break is a mistake. The tag exists for a reason, and it’s helpful for addresses or business hours, for example.)

Style

  1. Don’t include styling on your HTML document.

  2. One CSS file should hold all the formatting for your website. Don’t use a different CSS file for each HTML file.

  3. Declare all colors as RGB values, either using rgb(x, x, x) or #xxxxxx.

  4. Paragraphs (p) must be at least 1em. (Ideally exactly 1em. Or in other words, leave its size alone, respecting the users’ preferences.)

  5. If two pieces of information have the same hierarchy but should look different, use classes. (Don‘t make h3 and h4 have the same visual hierarchy but with different colors.)

  6. Don’t use <i>, <em>, <b>, or <strong> on a whole element or paragraph. Also, make sure you’re using them for the right semantic reasons. Use the <span> tag if it makes more sense.

  7. Don’t use all caps on your HTML file. Use the text-transform property on CSS instead.

  8. Don’t use px for anything but borders. For everything else use only % and em, as appropriate.

  9. Don’t use pt either. Use only em for font-size.

  10. Use % for width and use em for max-width and min-width. (In some cases, you may want to use em for width as well.)

  11. The float property should be used for sharing a line only. The element either goes to the left or to the right. (You should almost always use left, though.) There’s no float: center! If it makes it easier for you, use display: inline-block; instead.

  12. Avoid negative margins (or negative paddings). Use Eric Meyer’s Reset CSS tool if you don’t want to deal with default margins messing with your document’s style.

Classes

  1. Be as economical as possible with your classes. In other words, use as many classes as you need, but no more than that.

  2. Use semantic names for the classes. (i.e. “highlight” rather than “yellowbg”; “additionalinfo” rather than “rightcolumn”.

  3. “Container” is the div around the content of your page. The other divs should be named semantically.

  4. All class names should be lowercase and have no spaces. Also, they should not start with numbers and should be short (up to 12 or 15 characters).

Images

  1. Use the alt atribute on the image to be as descriptive as possible. Remember, that’s what users will read if they can’t see the image.

  2. Include a period in the end of all alt atribute text. It causes screen readers to pause slightly, making for a better reading experience. ➤

  3. You don’t need to include “picture of”, “image of”, or “photo of” in the alt attribute of the img tag. (“Hamburger on a plate.” rather than “Photo of a hamburger on a plate.”)

  4. All file names should be lowercase and contain no spaces.

  5. Keep all the images in your server.

  6. It’s good practice to keep the images in another folder. (Optional)

  7. Do not save photos as GIFs. Photos and rich illustrations should be JPEGs (or PNGs if transparency is needed).

  8. Don’t use the width and height attributes on the img tags. (Style them in the CSS, instead.)

Links

  1. Links within your website should be only the name of the file (e.g. index.html, contact.html, etc.), and not a big link to a file on your computer.

  2. Links within your website (internal links) don’t need full URL either.

  3. External links need full URL (http://www.example.com or https://www.example.com).

Troubleshooting

  1. If nothing on CSS seems to be affecting your page, make sure you have all the right curly brackets (}) in place.

  2. Another common problem is forget to include colons and semicolons on each property of the CSS. (property: value;)

  3. If it still doesn’t work, make sure you’re working on the right files. (Change some word on h1 to check if the HTML file is right, or give the body a background color to check it the CSS file is correct.)

  4. If you can’t upload files to the server, you may have run out of space. Cal Poly only gives us 50MB of space. Even though it’s not much, it’s plenty if you’re using images with appropriate size for the web.

Always check your work on multiple devices, browsers, and window sizes.
