GRC 338: Assignments

HTML/CSS website: content hierarchy

For this assignment, you will have to create a fictitious, small business to design a website for. It can be a restaurant, cafe, bakery, etc., as long it fits the requirement. (Keep it simple.)

Work on the content for the website’s four pages: Home, About, Menu, and Location. (Don’t worry, you’ll be able to use nice images, colors, and typefaces later. But for now, it’s only the content.)

You may borrow the content from other websites. For example, in the About page, you may have the story very much inspired by a restaurant that you know. Or maybe the menu is exactly from another place. Just make sure that the markup (the HTML structure) is all original.

Be careful: When you copy content, you may get some all-caps in the HTML, and you don’t want that.

Publish your pages on the URL:
username.github.io/grc338/website-content-hierarchy

Tasks

  1. Start with template for new HTML files.

  2. Download the initial CSS file and the placeholders for the images.

  3. Write the content for all the four pages. (Do not use placeholder content for your text.)

  4. Include gray rectangles as placeholders for all images (except for the logo). (I’ve included one placeholder file in the package, but you want to copy and rename it for every use.)

  5. Write the tentative alt attribute (no placeholder) for all images. Example: <img src="customers.jpg" alt="Customers eating at Bruno’s Restaurant’s patio facing Grand Avenue."

  6. Use a different placeholder for the logo. Again, write the alt attribute for the logo as in alt="Bruno’s Restaurant’s logo.", where you replace Bruno’s Restaurant’s with the actual name of the place you’re designing for. Don’t write alt="placeholder for the logo", because it defeats the purpose of a placeholder. Also, don’t name it logoplaceholder.png either, name the file as you’d name the actual logo image; e.g.:brunos-restaurant-logo.png. )

  7. Include the same navigation on all pages. You want to write your navigation menu as a list. (Also, you may be tempted not to include a link from a page to itself, but it’s not a good idea.)

  8. You don’t need a link to the home page on the navigation, though. Just make the logo a link to home and remove Home from the navigation.

  9. Check the list of mistakes to avoid.

  10. Don’t use any style.

Requirements

  1. Four pages with the tentative, final content.

  2. Menu with a total of at least twenty items, in at least three different categories. (That’s the actual menu for food, drink, etc., not navigation.)

  3. Each item on the menu needs description and price.

  4. At least one image per page (besides the logo).

  5. No styles whatsoever. Just keep the one selector and property I included in the CSS file.

Evaluation

For now, you will be evaluated on your HTML skills as well as on your hierarchy and semantic choices.

Use proper headings for headings, proper paragraph tags for each paragraph, appropriate list and list items tags for the lists, etc.

(Don’t worry about how your pages will render on the browser. “I don’t like the bullets” is not a good reason for not using lists; you can style the bullets out in CSS next week.)

Penalties in the list of easy mistakes to avoid are 20 points now. There’s no reason not to avoid all of them, though.

Tips

  1. Keep the link from all of your pages to a single CSS file, so you don’t have to do it later. But keep the CSS as is now.

  2. Don’t forget the navigation on all pages.

  3. Make the logo a link to home. (Then you won’t need a “Home” link on the main navigation.)

  4. Create a different file for each of the gray rectangles. Then, name them properly, so you can easily replace the image file later, without changing the HTML. (Name the placeholder files with names such as storefront.jpg, hamburger.jpg, etc., rather than placeholder1.jpg, placeholder2.jpg, etc.

  5. And don’t use any styles.

Submission

  1. You don’t have to submit anything on Canvas.

  2. When you’re done, use tools such as FreeFormatter, HTML Formatter, or Code Beautify to organize your HTML code.

  3. Go to your public repository on GitHub, and open a new folder named grc338/website-content-hierarchy/. ➤

  4. Create a new index.html file and commit. ➤

  5. Upload all four HTML files (index.html, about.html, menu.html and location.html) to your GitHub repository. ➤

  6. Upload your blank CSS file too, to avoid the error.

  7. Upload all your placeholder image files too.

  8. Click on your name to check if it’s working.
