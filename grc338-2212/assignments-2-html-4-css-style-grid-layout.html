CSS style: grid layout

Using the files from the last assignments, use CSS Grid to make the page look good on laptop and desktop screens.

Tasks and requirements

  1. Read all the instructions carefully before starting.

  2. Start by reading my comments from the last assignment. Make sure to address every issue on it.

  3. Download the new HTML file.

  4. Copy your CSS and JPEG files from last week into the new folder.

  5. This is what your page should look like on the browser.

    ★

  6. You want your content centered on the page.

  7. You want title, image, and captions take the whole width of the content.

  8. The Ingredients and Directions sections should take one column each.

  9. The footer must take the whole width of the content.

  10. Check the list of mistakes to avoid.

  11. Keep your CSS organized.

  12. Publish your pages on the URL: username.github.io/grc338/css-grid-layout

Evaluation

I’ll be assessing your ability to create a solution that is visually interesting and appropriate for the content, but you’ll be mostly evaluated on your technical skills using CSS.

Tips

  1. To center your page on the window, give the div.container the properties max-width: XXem; and margin: auto;, replacing XXem with any value you want, probably somewhere between 30em and 50em for this layout.

  2. Since only the section elements will be side-by-side, their parent, the main element, will be the grid container.

  3. You may use Google Fonts and Adobe Fonts, but you have to use the @import link directly in your CSS file, because, you know, you can’t change the HTML file!

  4. Do not change the HTML file for any reason! Not one bit!!!

  5. Always make sure to avoid the most common mistakes

Suggestion

  1. Don’t forget: You want to work offline, saving your files on your computer. Only when you’re happy with the results that you want to upload to GitHub. You may change your mind and upload a new file, but what you don’t want is to work straight on GitHub.

  2. I suggest, however, that you upload the old HTML, CSS, and JPEG files to the new folder in GitHub before you even start changing anything.

  3. Then you check your name. It'll be the last week's layout, of course.

  4. When you're done working with your new CSS, you use it to replace the old one.

This will prevent you from emailing me on Monday night saying that your file is not working.

Whatever you do, though, you may not edit the HTML file at all.
