CSS style: grid layout

Using the files from the last assignments, use CSS Grid to make the page look good on desktop screens.

Tasks

  1. Start by reading my comments from the last assignment. Make sure to address every issue on it.

  2. Copy your files from last week into a new folder.

  3. Sketch your layouts with pen and paper.

  4. Include on the media query only what’s absolute necessary. (i.e.: only what’s different on wider screens.)

  5. Check the list of mistakes to avoid.

  6. Keep your CSS organized.

Don’t forget: You want to work offline, saving your files on your computer. Only when you’re happy with the results that you want to upload to GitHub. You may change your mind and upload a new file, but what you don’t want is to work straight on GitHub.

Whatever you do, though, you may not edit the HTML file at all.

Publish your pages on the URL: username.github.io/grc338/css-grid-layout

Requirements

  1. You’re not required to use all the properties we’ve covered. Use as many or as few as you find appropriate.

  2. I may have overdone it with divs and classes. Don’t feel like you need to style every single class I used in my HTML file. You may choose to ignore some of them if you think it’s the most appropriate.

  3. Have I mentioned that you are not allowed to change the HTML file?

Evaluation

I’ll be assessing your ability to create a solution that is visually interesting and appropriate for the content, but you’ll be mostly evaluated on your technical skills using CSS.

Tips

  1. If you want to center your page on the window, give the div.container the properties max-width: XXem; and margin: auto;, replacing XXem with any value you want, probably somewhere between 30em and 80em.

  2. You may use Google Fonts and Adobe Fonts, but you have to use the @import link directly in your CSS file, because, you know, you can’t change the HTML file!

  3. Do not change the HTML file for any reason! Not one bit!!!

  4. Always make sure to avoid the most common mistakes

Hint

  1. I suggest that you upload the old HTML, CSS, and JPEG files to the new folder in GitHub before you even start changing anything.

  2. Then you check your name. It'll be the last week's layout, of course.

  3. When you're done working with your new CSS, you use it to replace the old one.

This will prevent you from emailing me on Monday night saying that your file is not working.

Submission

  1. You don’t have to submit anything on Canvas.

  2. Got to your public repository on GitHub, and open a new folder named grc338/css-grid-layout/. ➤

  3. Create a new index.html file and commit. ➤

  4. Upload all three files (index.html, lasagnastyle.css, and lasagna.jpg) to your GitHub repository. ➤

  5. Click on your name to check if it’s working.
