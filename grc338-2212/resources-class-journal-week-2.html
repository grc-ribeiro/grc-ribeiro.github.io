I’m enjoying the textbook more than usual. Maybe its my AP English kicking in, maybe its just zoom burn, but I genuinely enjoyed and understood the terms this week.

I knew about the idea of functional consistency before but didn’t really know how to put words to it. I think I want to look into it more and see how I can use it in my own work to make it easier to view.

It happens a lot with the principles. You’re now putting names in concepts you intuitively knew. It’ll help you be more conscious about them in the future.

I learned to distinguish the difference between consistency and similarity. Consistency is having parts that are expressed in similar ways so that the system is more learnable and usable while similarity helps show the relationship between elements.

You are correct.

I never knew that the control symbols for music and video players were an example of functional consistency because I always overlooked that aspect of design. Functional consistency allows people to use existing knowledge about how controls function so they can learn new devices much easier.

Very much true. And some solutions are so often seen that we don’t even notice them anymore.

It would be interesting to investigate the use of intentional inconsistency, particularly between platforms/devices. Should a website offer the same exact experience on desktop as it does on mobile? Or could a completely unique experience be offered, one that plays to the strengths of the platform?

You don’t want desktop and mobile exactly the same. Consistency doesn’t mean uniformity.

One thing that fascinates me is how intertwined these (and many other) design principles are, and how essential they are to creating effective designs. If you want to make two elements in your design feel related you’re likely using similarity of shape/size/color, proximity, and some kind of alignment to accomplish that. If you’re only utilizing one principle, your design won’t be nearly as effective as using multiple.

That’s very true.

Something I hope to learn more about is how to combine all these elements to create a well designed website that is efficient and successful to clients.

That’s the goal.

Elements with the same hierarchy should have the same tag.

Please don’t forget that.

How critical is the organization of your HTML code to the final product?

It depends on what you call the final product. It doesn’t make any difference for the browser. But it makes total difference for any human being (including you and some who’s reading it, like I will).

I am feeling very excited to learn more about HTML although still very nervous.

Don’t be nervous. It’s easier than you think. I’m glad you’re excited, though.

I enjoy how thorough you are at teaching HTML, breaking down each part and showing examples has been making this learning process much smoother.

We’ll go step by step. If you learned HTML before, please forget your hacks as well as your fears and annoyances.

I learned the importance of disregarding aesthetics when assembling code in HTML, because it’s much more important to create a well-functioning system and then reassemble the design afterwords in CSS.

I’ve learned that HTML isn’t as complicated as it appears.

Remember that HTML is not for styling, it’s for the basic structure. We can make things look pretty with CSS, but we make them work with HTML.
