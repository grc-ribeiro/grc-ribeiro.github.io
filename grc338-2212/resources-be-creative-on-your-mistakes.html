Be creative on your mistakes

Here’s how you can avoid the ones everybody else makes.

Basics

  1. All your content should be in the <body> of your document. 

  2. Close all the tags you opened. (Don’t forget to close <body> and <html>.) ➤

  3. All the pages need to have a <title> in the <head>. ➤

  4. All quotes in the code (not in the content) need to be straight quotes ("" not “”). It won’t work otherwise. (If you copy your code from another program, it’ll likely come with “smart quote” rather than the straight quotes. And it will break your code.) ➤

  5. In the content, however, use proper apostrophes, quotes, and dashes. ➤

  6. Your homepage file needs to be named index.html or you will get an error. ➤

  7. You can’t just make up HTML tags. Please, check the list of all valid HTML elements.

Type hierarchy

  1. Pieces of text with the same hierarchy should have the same tag. (p, h1, h2, etc.) You may, for example, use many <h2> tags throughout your page. ➤

  2. It never makes sense to have a <p> tag inside a heading tag (<h1>, <h2>, etc). ➤

  3. There’s no p2. ➤

  4. There’s no h7. ➤

  5. List items need to live inside lists. ➤

  6. Only list items can be nested inside a list. (Nested inside the list item, though, you may have other elements, such as headings and paragraphs.) ➤

  7. Avoid line breaks (<br />). Use lists or new paragraphs and style them properly. (That’s not to say that any use of line break is a mistake. The tag exists for a reason, and it’s helpful for addresses or business hours, for example.)

Style

  1. Don’t include styling on your HTML document. ➤

  2. One CSS file should hold all the formatting for your website. Don’t use a different CSS file for each HTML file.

  3. Try not to make different styles for each page either. Your goal is to make them as consistent as possible.

  4. Declare all colors as color values, using rgb(x, x, x) or #xxxxxx, or hsl(x, x%, x%). Don’t use the color names. ➤

  5. Paragraphs (p) must be at least 1em. (Ideally exactly 1em. Or in other words, leave its size alone, respecting the users’ preferences.) ➤

  6. If two pieces of information have the same hierarchy but should look different, use classes. (Don‘t make h3 and h4 have the same visual hierarchy but with different colors.)

  7. Don’t use <i>, <em>, <b>, or <strong> on a whole element or paragraph. Also, make sure you’re using them for the right semantic reasons. Use the <span> tag if it makes more sense. ➤

  8. Don’t use all caps on your HTML file. Use the text-transform property on CSS instead. ➤

  9. Don’t use px for anything but borders. For everything else use only % and em, as appropriate.

  10. Don’t use pt either. Use only em for font-size.

  11. Don’t use negative margins (or negative paddings). Use Eric Meyer’s Reset CSS tool if you don’t want to deal with default margins messing with your document’s style.

  12. Use margins and paddings to clear area around content only. (If you use margins bigger than 3em or 5em, it’ll probably break the layout.) ➤

  13. Use % for width and use em for max-width and min-width. (In some cases, such as a logo, you may want to use em for width as well, but you usually don’t want it.)

Classes

  1. Be as economical as possible with your classes. In other words, use as many classes as you need, but no more than that.

  2. Use semantic names for the classes. (i.e. “highlight” rather than “yellowbg”; “additionalinfo” rather than “rightcolumn”.

  3. “Container” is the div around the content of your page. The other divs should be named semantically.

  4. All class names should be lowercase and have no spaces. Also, they should not start with numbers and should be short (up to 12 or 15 characters).

Always check your work on multiple devices, browsers, and window sizes.
