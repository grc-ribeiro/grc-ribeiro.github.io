Restaurant menu with style

For this assignment, you will style your page from last week, without changing its semantics.

You can change the style of anything. Change colors, type size, margins, etc. Try to make it look as good as possible.

But again, keep your HTML semantic.

Tasks

  1. Start by reading my comments from the last assignment. Make sure to address every issue on it.

  2. Get your HTML file from last week and link it to a new CSS file.

  3. Style the restaurant menu using the CSS file, making sure it looks appropriate on a smartphone. (You don’t need to worry about other screen sizes for this assignment. It’s smartphone only.)

  4. You can make your browser window really narrow, to avoid worrying about other sizes. Even better, you can use the mobile built‑in simulator, also called User Agent, in your favorite browser.

Don’t forget: You want to work offline, saving your files on your computer. Only when you’re happy with the results that you want to upload to GitHub. You may change your mind and upload a new file, but what you don’t want is to work straight on GitHub.

Requirements

  1. You will need these lines into your HTML file,  between the <head> and the </head> tags: 

    <meta name="viewport" content="width=device-width, initial-scale=1.0">  
<link rel="stylesheet" type="text/css" href="style.css">

    (I’ve been doing web design since the early 2000s, and I don’t think I’ve ever typed the lines above. I always, always, always copy and paste them. Why wouldn’t I?)

  2. You may make changes in your HTML to apply classes to elements or to span tags, but you don’t have to. If you do use classes, though, make sure to name your classes semantically (i.e. appetizers or price, never greentype or italic).

  3. Don’t use the b, i, em, or strong tags on your HTML. Don’t use any other tags to change the style in your HTML. And don’t use the style attribute in your HTML tags either. (If you don’t know what it means, don’t worry.) Keep all the style in the external CSS file.

  4. For this assignment, use only serif or sans-serif for the font-family property.

Evaluation

Beyond your technical skills on the HTML and CSS files, you will be evaluated on your capacity to apply appropriate visual language to the content, respecting its visual hierarchy.

A common mistake is not to use the proper markup for styling reasons.“I didn’t want indentation” or “I didn’t like the bullet points” are not valid reasons not to use lists or list items. Please check again a few tips about lists. And if it’s a list, keep the proper list and list items tags.
