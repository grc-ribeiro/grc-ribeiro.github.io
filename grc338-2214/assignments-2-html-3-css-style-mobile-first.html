GRC 338: Assignments

CSS style: mobile first

I’m providing the structure for a webpage and your job is to style it to the best if your abilites. Think about some of the design principles we talked about (e.g. concistency, color, highliting, legibility, etc.) and make the page as interesting as possible. I have started the CSS with very few lines; feel free to use them or not.

I also included a placeholder image. You need to replace it with an appropriate image.

For this assignment, the goal is to make it look good on smartphone only. You don’t need to worry about tablets, laptops, and desktops. (We will work on it next week.)

Whatever you do, though, you may not edit the HTML file at all.

Publish your pages on the URL: username.github.io/grc338/css-mobile-first

Set up

  1. Download this zip file.

  2. Unzip the file. It contains one HTML file, one image, and one CSS file.

  3. Fine a nice, appropriate image on the web.

  4. Open the CSS in Brackets.

Tasks

  1. Replace the image, without changing the HTML file.

  2. Check the list of mistakes to avoid.

  3. Work offline on your CSS file, until you found a solution that you’re happy with, but don’t change the HTML file.

Requirements

  1. You’re not required to use all the properties we’ve covered. Use as many or as few as you find appropriate.

  2. I may have overdone it with divs and classes. Don’t feel like you need to style every single class I used in my HTML file. You may choose to ignore some of them if you think it’s the most appropriate.

  3. Have I mentioned that you are not allowed to change the HTML file?

Evaluation

I’ll be assessing your ability to create a solution that is visually interesting and appropriate for the content, but you’ll be mostly evaluated on your technical skills using CSS.

Tips

  1. Remember, it’s smartphone only. Don’t even think of having two columns.

  2. You may use Google Fonts and Adobe Fonts, but you have to use the @import link directly in your CSS file, because, you know, you can’t change the HTML file!

  3. Do not change the HTML file for any reason! Not one bit!!!

  4. Always make sure to avoid the most common mistakes

Submission

  1. You don’t have to submit anything on Canvas.

  2. Got to your public repository on GitHub, and open a new folder named grc338/css-mobile-first/. ➤

  3. Create a new index.html file and commit. ➤

  4. Upload all three files (index.html, lasagnastyle.css, and lasagna.jpg) to your GitHub repository. ➤

  5. Click on your name to check if it’s working.
