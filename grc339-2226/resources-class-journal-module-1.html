Can flexbox and CSS grid be used interchangeably or should they be used for different things?

Good question, and the answer is yes. Many times, they can be used interchangeably. However, there are some things that are easier to achieve with one or another. A (rather vague) rule of thumb is that you want to use grid when you want to have control of the layout and flexbox when you want the content to control the layout, giving it more flexibility. It’s okay to be confused by this statement for now, though. Eventually it may click. You will be able to decide for yourself once you have more control of both.

It would be best to use flexbox when you want the content to shape the layout while using the grid when you want your layout to shape your content.

Very well put.

If you need to make changes to your code after you already uploaded your files to GitHub, you can click into the file you need to edit and press the trash can to the top right corner to delete the file. Then, you simply just upload your updated file to the assignment folder again.

You can also upload to replace or click on the file and edit it. But the trick on how to delete a file is the most important here. (I’m constantly asked about it.)

Is there a tool to make sure that lines do not have more than X words (16 in this case)? Or is counting each line that looks long the only option?

You have to count lines for now. Later in your career, you won’t need to; you’ll know at a glance.

It is important to be aware of the line length on your website. A good line length should be between 8 and 16 words. To ensure your line length is not too long, we can define a max-width for the div container.

You can define a max-width for anything. For example: sometimes, I define my p with a max-width of 36em.
