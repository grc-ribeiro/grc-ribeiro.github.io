Sometimes making changes in your code and saving is helpful if you don’t know what it does. Change a value and see what it changes, that way you can go back and set it as you want.

A comment can be indicated in CSS by writing /* and */around the comment. However, it is different for HTML, so I found online that a comment can be written between ``.

Yes. And in both languages, VS Code will toggle comment with the ⌘ / shortcut.

<> are only in HTML and {} are only in CSS.

You’re correct.

I wonder why we only use px for borders.

To keep the borders crisp. You don’t want the thin line falling in between pixels.

In CSS, when you want to change the color, don’t use the name of the color, instead use the HEX code.

One of the most common mistakes, believe it or not.

Margins bigger than 3em are going to be a mess.
