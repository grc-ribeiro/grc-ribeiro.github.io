GRC 338: Resources

Class journal, module 9

This class has gotten me in the habit of ctrl-s ing my work every few minutes. I keep instinctively doing it on Adobe applications which auto save nowadays but it’s still a good habit I think!

It doesn’t hurt.

I look forward to not using very bright and bad color pairings on our personal website. But I will also use this in the future as it does help point out what you are affecting on a website.

It really is helpful.

If I am confused on what I am supposed to change, I look back on my HTML to see what selector it was in.

That’s the only way.

Sketch out the grid if you are stuck and need to visualize the organization.

That’s a good idea.

So is it best practice to design on your smallest screen first, and then work your way up to the bigger screens?

Yes. Mobile first.

When I search up questions about CSS, many times Google will recommend something called a flex box for centering elements vertically and horizontally. What’s a flex box and why don’t we use it in this class?

It’s another way to set the page layout, instead of in conjunction with CSS Grid. We use it in GRC 339.

Are there any other ways other than using media query to adjust the responsiveness of a website?

Yes. If you’re using Flexbox rather than CSS grid, you may not need to rely so much on media queries.

Both ems and percentages are units that help layouts adapt to different screen sizes and user preferences, but we want to use em and rem. Only use percentages for the width of img tags.

Please.

In the assignment, why did we use inline instead of a grid to line up the navigation buttons?

First, because it’s easier and more straightforward. Also, you don’t know what’s the size of each column. (You could use auto auto auto, but it does feel like an unnecessary hack.)

Inside the media query goes anything that needs to change in wider screens.

Precisely.

Media query is useful when you want to modify websites based on a device’s parameters.

That’s correct. And it’s not only for width. For example, this very page change its colors depending on the user’s preferences for light or dark mode. This is done with CSS and this media query: @media screen and (prefers-color-scheme: dark).

Speaking what you need to be done out loud makes problem‑solving easy.

True. (They’re talking specifically about needing a maximum width and realizing they needed to use the max-width property.)

There is no declaration to make something a grid item.

Correct. Just make its parent a grid container.

I was curious if we were able to incorporate a hover style in our final webpage? I liked the color changes of the selected nav section but I’d like a hover style for my final website.

Yes, you can. It’s pretty simple too. Use nav ul li a:hover { } as the selector.

In GRC 339, will we use any other web development tools aside from HTML and CSS?

No. But in GRC 340, you can learn about JavaScript.
