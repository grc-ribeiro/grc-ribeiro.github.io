I learned that it is important to test your website multiple times to make sure that everything works. If I were in an in-person setting, I would ask my friends and peers to click around my website and make sure everything works.

And I’d have required it actually.

A question I have is how to make the checkbox so that you can only select one answer.

You can’t. A checkbox is supposed to take multiple answers. For only one answer, you need radio buttons.

From this class, I learned I prefer working with grid over flex. It’s easier for me to visualize, but I’m glad I know how to do both.

I find it easier too.

Grids and Flexbox have definitely been a learn-by-doing experience for me. They’re easier and easier every time I attempt it.

If they’re getting easier, that’s good.

One of the biggest issues I encountered in the last assignment was getting the input { display: hidden; } to work. If I simply just said that, the photo gallery page would function correctly but the newsletter would not. I ended up adding a div class around the main image of the photo gallery and adding the line .gallery input{ display: none ;}to my CSS file as a workaround.

Except that this is not a workaround, but the proper solution.

The gallery wasn’t working for half the photos which was frustrating, but after a night’s rest, we realized we missed some commas. Details are important!

Happened to several of you.

When you make an item a flex container, only its direct children objects become the flex objects. If there is a div or other section in the way, that will become the flex object instead of the items you intend.

Very true. It’s an easy mistake to miss.
