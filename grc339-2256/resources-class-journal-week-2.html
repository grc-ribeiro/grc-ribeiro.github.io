GRC 339: Resources

Class journal, module 2

I am slowly learning that learning how to code online is difficult. I need things to be explained to me in-person so that I (or other students) can ask questions in real-time and collaborate with other students in the class. I don’t know how CS majors are able to learn coding online so quickly.

It is a little trickier. That’s why I have the in‑person labs during the year. The summer, online version of the class is intended to those who are more comfortable with code and need to accelerate their education for some reason.

This week’s assignments were a bit more challenging, so I made sure to follow along with the video tutorials to help!

That’s the idea!

I would be curious to gain access to actual website’s HTML/CSS, to compare and contrast. I would love to pick up some of their design habits.

On every browser, you can look the source code of every page. That was extremely helpful when websites were more static. Nowadays, the code is way more complex when you peek at them.

Some list items in my form didn’t look exactly like the example given. I’m assuming most of that is based on your browser (e.g. Firefox vs. Safari)?

Yes. Rendering in different browsers (or different versions of browsers) are not exactly the same. And that’s something we always need to keep in mind. Test in different browsers, devices, and window sizes.

I realized that absolutely positioned elements don’t contribute to their parent’s height, which explained why the main element collapsed without extra padding or min height.

Exactly.

Is the functionality and animation of the CSS gallery something that can be done in JavaScript? If so, which is preferred in the industry?

The gallery can be done in JavaScript, and that’s probably what most people would do in the industry. The non-JS version that you learned can be more accessible, secure, and faster, though. In the industry, you may end up working where they rely heavily on JavaScript or maybe you will work at a place where they despise it. That’s why it’s good to have some experience with JavaScript (GRC 340) and experience with replacing it with pure HTML/CSS.

Used Flexbox on grouped form sections (.order ul, .method ul) so options align neatly in rows and wrap when needed.

That’s a nice trick to keep it consistent.
