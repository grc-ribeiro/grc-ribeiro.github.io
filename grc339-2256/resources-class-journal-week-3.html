Class journal, module 3

It’s interesting that in web design, we design for mobile first and then adapt it to bigger screens. In UX design, it’s usually the other way around - we prototype for desktop first and adapt it for mobile later on. Just an observation.

OK. Several things to unpack here:

  1. Web design is very much part of UX design.

  2. Sometimes in web design, we design for desktop first; it depends on the main use. For example, Canvas is probably designed for desktop first. Conversely, many apps, such as message apps, are designed for mobile first.

  3. Furthermore, you seem to regard what you learned from one professor (me) as an absolute truth in web design. That’s a common trap in education: You think that whatever you learn first is necessarily right, and everybody who disagrees with your first teacher is wrong. And it’s just not true. If in some point you find someone designing for desktop first and mobile later, they may have their reasons. Disagreeing with me doesn’t make one wrong.

Sometimes I notice margins I don’t want in certain places, and I’m not sure what element the margin is on. I use the web developer tool through the browser to find which element I should be looking at to change the margin in CSS.

Yes, the inspector is extremely helpful.

I find w3schools.com to be really helpful when I get stuck on something.

And MDN too.

A challenge I had was applying the margins to h2 and the paragraphs to match.

Use rem instead of em, then. (Look it up.)

Make sure to properly name your classes in your HTML and then identify it accordingly in your CSS. There were multiple times nothing would appear or change because I didn’t realize I wrote “secondary” in my HTML and “secondary-story” in my CSS.

That is, indeed, a common mistake.
