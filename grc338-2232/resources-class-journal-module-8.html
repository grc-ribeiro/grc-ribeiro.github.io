I learned about the picture superiority effect. I already understood and often implemented the concept in my designs but now I know there is a name for it.

That’s often the case with the design principles. They give you names and reasoning for decisions that you used to make based on your (often right) gut feelings. Now you can be more confident on your decisions and explain them later.

Asking questions really helps me even if I know I can consult with Google. Google may give me information that is too complex and ends up giving me more stress.

I also learned that it is important to add a period after your alt text in <img> tags because it gives the automatic reader programs the chance to pause, thus helping the person trying to understand the website.

That’s true. Please don’t forget the period in the alt attribute.

The <img> tag doesn’t have a closing tag- I wonder if there are other tags that also don’t have a closing tag or is <img> the only one?

<br/> and <hr/> are two that come to mind now. And that’s why some like to add the (optional) / at the end of the self-containing tags (that must be empty). But <br>, <br/>, or <br /> are very much the same in HTML.

For rules, when items are not separated by a comma it only applies to the last one that is contained within everything else.

Precisely.

I remembered that width values should be in %, and max-width values should be expressed in em.

Most of the times, yes. Sometimes, you want width in em. But I can’t think of an example in which max-width in % would make sense. (I’m sure that there’ll be some complex layouts where it would be helpful, but not for this course.)

I was surprised that the media query only had 7 declarations at first, but when setting declarations outside of the media query that had properties such as max-width, it made sense. Those values were only shown at certain widths which was not applicable to the smaller screen from last week.

That’s exactly right.

I’m a bit confused about when the margin is auto when there is not another value defined, what is it automatically set to? It worked though, just need help understanding it.

From the excellent W3Schools: “You can set the margin property to auto to horizontally center the element within its container. The element will then take up the specified width, and the remaining space will be split equally between the left and right margins.” It is automatically set to the half of the remaining horizontal space to each side.

The nomenclature for CSS is pretty hard to remember sometimes.

It’s ok. You’ll get there.

Using auto for the grid columns and rows makes it a lot simpler.

It does. repeat(2, 1fr) is a good way to write it too.

When centering div containers so that they will always be centered in the window, a width must be given in percent and the margin property must have the value auto.

Nope! You want a max-width in em. But you’re right that the margin property does get the value auto.

When should I use “em” versus “rem"?

Most of the times, it won’t make a difference. But if you always use rem, it tends to be easier. I use em because it’s a habit. (Although R.E.M. is probably my favorite band ever, but that’s not relevant.)

You can just say .container instead of div.container.

Correct. And if you are only applying the class to one kind of element, it won’t make a difference. I like to write element.class, however, because it helps me keep track of my own CSS rules. But it’s a matter of personal preference and not necessarily best practices.

Media queries are kinda cool.

Aren’t they? I think so too.
