You’ve always taught us to name the HTML file for our website homepages “index.html.” I’m curious if there’s a particular reason why, or is it more of a common naming convention?

It doesn’t work otherwise. From this answer in StackOverflow:

When you arrive a website, for example www.website.com, you're not pointing to a file (like you would be if you typed www.website.com/about.html), you're pointing to a directory listing of all the files.

The webserver will try to serve a file, typically called index.html or index.php by default, but it could be something different, and it's configurable by editing your webserver's config files.

If the server doesn't find any file to serve (because you didn't include an index.html file or because you renamed it without editing the server's config) you will see a listing of the files, which is rarely the desired behavior, especially at the root of a website.

You could configure your webserver to something else, although if it’s possible in GitHub. But I don’t think it’s worth it.

A topic that I would love to investigate more is whether the sequence in which I write my CSS code influences how it’s displayed on the website. (Especially for the div.container).

It does. Let’s say you write color: #f00; and then color: #00f; for the same selector. Whatever comes last overrides what came first. With that said, though, more specific also overrides less specific even if they come first. I always try to write my CSS code from more generic to less specific to avoid different results in different browsers.

Why are rows the default for flex containers and how do we change it into columns?

Why? I don’t know. Probably because it’s the most common case. In the excelent CSS-Tricks, we can find A Complete Guide to Flexbox. In the Flexbox properties, I can see that the flex-direction property acceppst the row, row-reverse, column and column-reverse values. So if I want to change my flex container direction into columns, I write the declaration flex-direction: column;.

I am a little confused about what flex-grow is exactly doing.

It tells the item that it should grow to take the available space beyond its initial size (defined by the flex-basis property).

Flex-wrap wraps items if necessary” — Can you clarify when it is “necessary"? When the element extends beyond the viewport?

If the elements extend beyond their containing element size. Let’s say the flex-direction is row (default). Then, if the sum of all flex items’ widths is wider than the flexbox width, then some of the items will move to the next lines. This is probably what you want.
