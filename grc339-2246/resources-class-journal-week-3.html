It’s better to use Firefox when looking at small screen sizes because Chrome behaves a little differently.

I’ve been telling you all to use Firefox for a while. I don’t even use Firefox as my main or secondary browser. But it’s the best for web development and the one that I use to look at your work when I’m grading.

Even though this class asynchronous, it’s still good practice to take breaks when I get frustrated with my code.

Always. If it’s not working, take a break and come back later.

I’m excited to start working on the personal digital magazine project but a little nervous about developing a brand voice for it.

Make sure to use good references. Who has a similar brand voice? What’s their visual strategies to convey such voice? You want to be very intentional in all of your design decisions. “Don’t try to be original, just try to be good.”

This helped me understand media queries better: CSS Media Queries Guide | CSS‑Tricks.

Thanks for sharing, Riley!

After talking to Bruno, I learned that using min-width in a media query is preferable to using max-width (I tried it myself and found this to be true).

Always go with mobile first.

Another habit that I will be trying to avoid on the next projects is to not just copy and paste the CSS code into the @media query then adjust the code because as Bruno noted, we don’t need anything in the media query that’s not different from the other screen sizes.

You sure don’t.

How do you make the inline display for a list to appear vertical in smaller screens?

Ideally, you want to put the display: inline; declaration only in the media query, which should be applied for the larger screens. If you do the media query for the smaller screen (which is not what I want you to do), you need to to do display: block; for the nav.ul.li in the media query. Again, it’s possible, but not what I recommend.

Working with mainly CSS for the past few weeks, it’s always good to try to simplify the code as much as possible (i.e. decreasing redundancies).

It really is.
