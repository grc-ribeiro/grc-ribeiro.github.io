To remove background color from an element, why does background-color: none not work but background: none work?

You can define a background color with either the background or the background-color properties. By default, they’re both none for all the elements. If you need to remove a background color, it’s because you added it before. Then, to override what you declared before, you need to use the same property. So, for example, if you gave all h3 a background color using the background-color property, but you want to remove the background for h3.someclass, the background: none; declaration will not do anything, since the default value for background is already none. You need to use the same background-color property you used for adding the color in the first place.

Why is it that you can change the typeface of the navigation menu links using the nav ul li selector in CSS, but the color must be changed using the nav ul li a selector?

You could change the typeface on nav, nav ul or even nav ul li a too. But colors are different because the anchor element (<a>) has default colors (blue for unvisited and purple for visited links on most browsers). If for example, you were changing the color of list items that are not links, nav, nav ul or nav ul li would work just like when you change typefaces. But because it’s a link inside the list item, you need to be specific.
