Due to the aesthetic-usability effect, are aesthetics more important than usability? Or should you aim to strike a balance between both? I’m a little confused on that.

You want both. And it’s not a balance, because aesthetics help usability. That’s the aesthetic‑usability effect.

If a system is designed well enough and has very strong error prevention, do you need to include error forgiveness or vice versa?

It depends. They’re both helpful. Error forgiveness is always more desirable, and error prevention can be annoying. (Imagine if Photoshop asked you for confirmation of each action instead of offering undo.) If an action is fully and easily reversible, you don’t need error prevention to be so strong. At the same time, one way to prevent errors is to design systems that are very intuitive, so users don’t need to keep reversing their actions all the time. Trial and error can get old after a while.

I am still trying to understand why you have to put the content inside a container div if it only includes everything in the body.

Good question. We do this so we can style our entire content. For example, with the lasagna page, we used the container div to center the content with a different background color. We could, in theory, apply the same properties that we applied to container (max width, background color and an auto margin) to the body and then give a different background color to the html. But some browsers used not to behave well with max-width applied to the element body. Would it be safe to ditch this old habit now, since browsers got much better? Maybe. But that doesn’t seem to have been the general practice in the industry, at least not yet. And there’s no downside to using this non‑semantic element for styling.
