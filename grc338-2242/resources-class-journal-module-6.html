I did some research on why px should only be used for borders in our CSS syntax. px should be designated for fixed elements. em is a responsive unit that grants higher accessibility and increased responsiveness across devices, and therefore are more desirable for fonts. Here’s more information: https://elementor.com/help/whats-the-difference-between-px-em-rem-vw-and-vh/.

I have one article about it in module 3 too. The you linked too explains it well. But I’ve seen students doing the math trying to convert pixels to em. Don’t. Stop thinking in pixels. It’s like speaking a second language. When I speak English, I don’t think in Portuguese and translate to English, I think in English. It’s the same thing; we’re designing and thinking in em, not px.

For the grid, is fr the only unit we can use? Or are there other recommended ones?

Sometimes, you may use em and auto. So for example, grid-template-columns: 12em auto; will make the first column exactly 12em while the second column will take the remaining area.

I learned how to use the fr value although I am still a little confused on what it is.

It’s a fraction of the grid width. If you have grid-template-columns: 1fr 1fr 1fr;, there’ll be three columns with the same width. If you have grid-template-columns: 1fr 2fr 4fr;, there’ll be three columns with the second columns being twice as wide as the first, and the third being twice as wide as the second. How wide will they be? Let the computer do the math.

I think I am still confused and need to work on margins and paddings. I know the definitions but when putting it into practice there is still a lot of trial and error.

This is normal, though. There’s a lot of trial and error with margins and paddings.

It is easier and more beneficial to use fr values for the grid-column property rather than using percentages.
