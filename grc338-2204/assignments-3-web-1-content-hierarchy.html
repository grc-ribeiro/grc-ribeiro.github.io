GRC 338: Assignments

HTML/CSS website: content hierarchy

For this assignment, you will have to create a fictitious, small business to design a website for. It can be a restaurant, cafe, bakery, etc., as long it fits the requirement. (Keep it simple.)

Work on the content for the website’s four pages: Home, About, Menu, and Location.

(Don’t worry, you’ll be able to use nice images, colors, and typefaces later.)

Publish your pages on the URL:
username.github.io/grc338/website-content-hierarchy

Tasks

  1. Start with this template for new HTML files.

  2. Write the content for all the four pages. (Do not use placeholder content for your text.)

  3. Include gray rectangles as placeholders for all images (except for the logo).

  4. Write the tentative alt attribute (no placeholder) for all images. Example: <img src="placeholder.jpg" alt="Customers eating at Bruno’s Restaurant’s patio facing Grand Avenue."

  5. Include a different placeholder for the logo. Make it an image of the name in Helvetica Regular and the color black. (But write the alt attribute for the logo as in alt="Bruno’s Restaurant’s logo.", where you replace Bruno’s Restaurant’s with the actual name of the place you’re designing for. Don’t write alt="placeholder for the logo", because it defeats the purpose of a placeholder. )

  6. Include navigation on all pages. You want to write your menu as a list. (Also, you may be tempted not to include a link from a page to itself, but it’s not a good idea.)

  7. Check the list of mistakes to avoid.

Requirements

  1. Four pages with the tentative, final content.

  2. Menu with a total of at least twenty items, in at least three different categories. (That’s the actual menu for food, drink, etc., not navigation.)

  3. Each item on the menu needs description and price.

  4. At least one image per page (besides the logo).

Evaluation

For now, you will be evaluated on your HTML skills as well as on your hierarchy and semantic choices.

Use proper headings for headings, proper paragraph tags for each paragraph, appropriate list and list items tags for the lists, etc.

(Don’t worry about how your pages will render on the browser. “I don’t like the bullets” is not a good reason for not using lists; you can style the bullets out in CSS next week.)

Penalties in the list of easy mistakes to avoid are 20 points now. There’s no reason not to avoid all of them, though.

Tips

  1. It’s probably a good idea to go ahead and link all of your pages to a single CSS file, so you don’t have to do it later. But keep the CSS blank.

  2. Don’t forget the navigation on all pages.

  3. Make the logo a link to home. (Then you won’t need a “Home” link on the main navigation.)

  4. Create a different file for each of the gray rectangles. Then, name them properly, so you can easily replace the image file later, without changing the HTML. (Name the placeholder files with names such as storefront.jpg, hamburger.jpg, etc., rather than placeholder1.jpg, placeholder2.jpg, etc.

Submission

  1. You don’t have to submit anything on Canvas.

  2. Got to your public repository on GitHub, and open a new folder named grc338/website-content-hierarchy/. ➤

  3. Create a new index.html file and commit. ➤

  4. Upload all four HTML files (index.html, about.html, menu.html and location.html) to your GitHub repository. ➤

  5. Upload your blank CSS file too, to avoid the error.

  6. Upload all your placeholder image files too.

  7. Click on your name to check if it’s working.
