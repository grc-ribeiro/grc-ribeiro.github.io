GRC 338: Resources

Class journal, week 3

I learned about how to create a grid on a website. I also thought is was cool how you can use the inspector tool to see how grids are laid out on any given website.

The inspector on the browser is an extremely helpful tool. More helpful in Firefox than in other browsers.

I learned that when part of your code is red it means there’s something wrong with that tag.

Correct. I strongly suggest you to download Firefox. Open your HTML files in it and view the page source to check your code before you submit them. If any part of your code is highlighted in red, something’s wrong. (This is how I catch your mistakes, you should catch them before I do.)

One of the first things I realized was that visibility, or making a system’s state, available actions, and feedback easy to see, is very very very useful for usability by helping users understand what they can do with ease without guessing.

That’s correct! We can work on all of that in GRC 4900.

I recognized the importance of displaying clear visual feedback because visible controls and information inform users about what actions are available and at the same time they provide proof that their actions got done correctly.

Yes! Feedback is one of my favorite principles of usability.

Learning about the expectation effect made me realize that people’s opinions can be influenced before they even use a product, which is something designers need to keep in mind when testing and presenting designs.

And managing expectations is an important skill in life.

I learned that section should be used instead of a div when grouping related content because it is a semantic element. Non-semantic elements like div should only be used when there isn’t a more appropriate semantic element.

That’s correct.

I learned that using 1fr instead of percentages lets the browser do the math for the grid columns. This avoids layout problems when adding a column-gap.

That’s correct.

My next step is to spend more time practicing CSS layouts so I can recreate webpage designs more efficiently without relying on trial and error.

Twenty years in, I still rely a lot on trial and error. Undo is free.

This was one of the first assignments I actually had trouble with. I still don’t really know what was going on but when I would try to commit updates to my CSS on GitHub they wouldn’t complete and they were stuck in progress. I eventually got it to work but it came with a lot of deleting and re-uploading and refreshing and I think along the way some of my code got tweaked.

It happens sometimes. I don’t think it was your fault at all!

I’m curious how to make websites look different on web and mobile.

We’ll work on it soon. And it’s not that hard.

It is crazy how different a website looks when it’s the bare-bones HTML version, compared to after CSS styling is applied.

And we are doing it all over again.
