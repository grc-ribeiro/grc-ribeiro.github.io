Class journal, week 4

I feel like I understand responsive design a lot better after this module.

I’m glad to hear.

I’ve been having fun trying to make my CSS as accurate to the screenshots as possible, it’s always a great feeling to know I can look at the example and know how to basically replicate it on my own.

I’m glad that you feel this confident! Good job!

It was a bit harder trying to put together all of the different skills, instead of using them each individually. For the website, we have to think about classes and grids and hierarchy and links all at the same time.

Of course, it was challenging. But you all should be really proud how much you are able to accomplish in just four weeks of code. Speaking of which, I’ll send an anonymous survey after the class to ask you how you used artificial inteligence to help you. In different levels, I’m sure each one of you used it at least a little. Why wouldn’t you? But if it got to a point that it’s not challenging anymore, the instructor who will teach the new GRC 1090 in the semesters need to know that they can expect more from their students.

Organization has been really important in these most recent assignments, with all the different files and folders and copies I’ve been making and then moving them from VS Code to GitHub. It’s been really important that I keep my spaces organized and take everything one step at a time.

Absolutely!

I learned that every page on a multi-page website should share the same header and navigation so users always know where to find things no matter which page they’re on.

Yes. Consistency is super helpful for usability.

I learned that the face-ism ratio originated from research about gender bias in the media. It is a pattern I have recognized but not been able to name until now.

And we all need to be aware of these patterns. Sometimes, we make mistakes and follow patterns that we would prefer not to follow. When it happens, instead of being defensive or ashamed, we need to acknowledge our mistake, understand that it’s not a reflection of who we are, and try better next time.

I realized that in order for design to be good it must not only be appealing and pleasant but it must make it possible for a large number of people, In particular those with certain needs or limitations, to be able to use it. Also, the design must help people avoid errors and also make it possible for them to be able to undo such errors.

Exactly. That’s why we learn all these design principles, so we can make more informed decisions.

Learning about errors as a design principle made me think about all of the warnings and precautionary pieces included on packaging and within websites that we don’t usually notice, but that keep us from misusing the product or site. It must require a lot of experimenting to figure out where these errors usually occur and how to minimize them.

That’s a very good point. You need to pay attention to your users to help them avoid errors. There are some tests, but also observation, interviews, etc.

I have some ideas now for websites to make.

I look forward to seeing them!
