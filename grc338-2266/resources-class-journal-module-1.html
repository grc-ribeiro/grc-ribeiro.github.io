I enjoyed learning overall about HTML and codes, I have never worked with any sort of coding software/program and I have been really interested seeing how different functions within a code affect how they appear. I have been enjoying discovering that codes and HTML and all of these things that have always seemed way too complicated for me to learn are actually understandable once they are broken down.

I’m glad to hear you’re enjoying it!

I learned how to design basic webpages on GitHub.

Make sure you’re not designing them on GitHub. Use VS Code on your own computer and publish it on GitHub.

The “Entry Point” topic in our textbook reminded me of a trick I learned in a club on campus, iter8, that most (if not all) information should be accessible through no more than three clicks on the website. This ensures smooth navigation.

Not a real rule, though. Some things should be very easy to access; others can be tucked away as long as they’re easy to find.

Form follows function stood out to me because it showed that good design isn’t just about making something look nice and it’s about making sure every design choice has a purpose.

That’s a good way to summarize it.

The idea of an entry point was something I hadn’t really thought about before, but now I notice how every design needs an obvious place for users to start.

Most principles will be something that you never notice and now you will or things that you absolutely have noticed before, but didn’t know it had a name and was a design principle.

I previously didn’t know if the spacing in between lines of code had an effect on the actual website, and this week I learned that the spacing is only for organization and doesn’t have any effect on the website.

That’s correct.

I don’t think any topics remain confusing as far as what was covered in the videos/quizzes, however using GitHub as a beginner is pretty challenging.
