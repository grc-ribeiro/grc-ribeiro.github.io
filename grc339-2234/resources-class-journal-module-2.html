If you want side-by-side content to be double to size of the smallest content, you should have the flex properties of the biggest content to be double the size of the smallest content.

Correct.

From the video, I learned that the flex-basis sets the initial length of a flexible item.

Correct. And if you allow it grow but not shrink, it becomes the minimum width. It’s not wrong to use a value bigger than 0 for the flex-shrink property and to allow the element to shrink, though; I just find it much easier to think that way. But it’s my personal preference, not a requirement or even best practice. I’m only trying to help you make sense of flexbox and having more control of the elements by taking one variable out of the equation.

How can we set flexbox layouts to be columns instead of rows?

You could google that, but: flex-direction: column.

The values for the flex property should be entered in this order: flex-grow, flex-shrink, flex-basis.

Correct. And remember that there are no commas: flex: X X Xem.
