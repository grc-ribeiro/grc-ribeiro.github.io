I still am having trouble understanding the distinction between semantic and non-semantic elements. What did you mean when you were saying that semantic elements have intrinsic meaning?

They’re not only for styling. A header should be used for the header of the document. A ul is for an unordered list, etc. But the span was just to change the format. It’ll get easier to understand when we compare it with the div element that is non-semantic and will be used just to make visual (non‑structural) changes.

The header is always in the body section and is different from the head.

True statements.

When creating a mobile page, does changing the browser size affect the contents on the page?

It shouldn’t. The content should be the same.

I am still a little confused about where we should add margins as well as where both margins and padding should both be applied.

Try adding background to the element and see what difference it makes when you add margins or paddings.

I learned that our bullet points in CSS weren’t working because the ‘display: block’ element wasn’t written in the right area. What does this element mean in relation to the bullet list’s visibility?

The li element has the value list-item for the value display. When I mistakenly applied the value block to it in my interpretation of Eric Meyer’s, I broke its default format (with bullets and numbers for unordered and ordered lists, respectively). I need to clarify, though, that it was my mistake and not Meyer’s. Last year, a student (shoutout Sam Kim!) noticed that without the display: block; applied to the img element, sometimes there would be an undesired tiny margin under it. Trying to be clever, I included display: block; in my reset tool, and that’s how I broke the lists in my CSS. I have since fixed my general CSS reset line, but forgot to update in the zip file for this assignment. I’m not giong to pretend that I did this on purpose so you could learn from it, but there’s that at least.

I need to study on when to use %, px, em, and rem.

Use px for borders and only borders (well, and border-radius). Don’t use em; use rem instead because it’s still adaptive and relative, but much easier. And you probably want % only when you want the element to take the whole width of its containing element (width: 100%;). And it’s not that it’s wrong to use % for other cases or em instead of rem; but that’s how I use it, because I think it simplifies it. Therefore, that’s how I can make more sense of the matter to help you use it confidently.

Is rem the ideal unit to use, if not then what is the most consistent unit of measurement to use?

Yes. Use rem.

It seems like CSS has a lot of options depending on how detailed you want to be with styling (i.e. sans-serif family vs a specific font, “wide” vs a specific em/px space). Knowing these will be helpful for designing things the way you want or need.

That’s true. And you want to be specific. Use a specific font, specific values (in em or rem not px), specific color values too.

Padding can be specified to one side (i.e. padding-bottom).

So do margins.

How do you know which font-weight values to use? Does each font-family have a specific set of inputs for font-weight?

Yes, they do. Adobe Fonts and Google Fonts will let you know.

I am confused on what to use for font-weight. For example, you can input ‘normal’ or ‘300’. Is there a difference?

It depends on what the font asks for. Adobe Fonts (or Google Fonts) will tell you what to use for each font.

I think I am much more comfortable using padding, font-families, and text properties. Also using web safe fonts was much easier than I thought.

There’s no such thing as web safe font. Use fonts from Adobe or Google instead.

One thing I want to remember is to try not to have more than 10 to 12 words per line for legibility.

Yes, please! That’s very important for readability. More than you’d think.

They do. It’s pretty much the same idea of keeping the structure and hierarchy of the document.
