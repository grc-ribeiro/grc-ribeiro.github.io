A stronger pedagogical approach is to have students develop one product from concept to validated prototype over the entire semester. Each lab deliverable becomes a milestone that feeds directly into the next, mirroring an industry UX process. Below is a revised sequence with cumulative deliverables.

User Interface Innovations

Semester Project: Design an innovative digital interface that addresses a real-world problem (e.g., education, healthcare, sustainability, entertainment, civic technology, accessibility, or commerce).

Students will build the project incrementally over 15 weeks, submitting revised versions of previous work as they receive feedback.

Why this structure works

Instead of producing isolated assignments, each week's deliverable is a required component of the final UX portfolio. Students continuously revise earlier work based on instructor and peer feedback, closely following an industry-standard UX workflow:

Project Proposal → Competitive Analysis → User Research → Personas → Scenarios & Journey Maps → Information Architecture → Wireframes → Mid-Fidelity Prototype → Usability Testing → Design System → High-Fidelity Prototype → Ethical Review → Final Refinement → Professional Presentation.

This scaffolded approach ensures that every lab contributes directly to a polished, portfolio-quality project by the end of the semester.
