GRC 339: Resources

Class journal, module 9

Code Beautify is so satisfying to use. Every time I think my HTML is already organized, this website makes it look ten times cleaner.

If you find organized HTML satisfying, you’re my people.

I copied the HTML from previous labs to create the gallery and newsletter pages. For the styling, I found it easier to reference rather than copy the previous labs since there were a lot of styling differences.

That was the right approach.

Filtering google images using the tools dropdown makes it easy to grab a high resolution image in a pinch.

Nice tip!

I edited all of my photo gallery images in Photoshop to make them the same size, which I think definitely helped make a clean layout, but I was wondering if there is a better/more efficient way?

There are other solutions, not necessarily better or more efficient.

My next step is to visualize my finished product by creating a Figma prototype of what I want my finalized design to look like, and translate that to HTML.

I advise against it. Design on the browser is my suggestion.

Creating classes is very helpful for bigger webpages to keep everything organized and not override styles made for different sections.

It’s imperative. I mean, not literally, but yeah…

I noticed that my footer was hanging in the middle of the newsletter page. Something that fixed that was setting the position to relative and min-height to 100vh in the container, and making the footer’s position absolute and bottom at 0.

Nicely done.

It is important to organize your CSS in a way that makes sense for you to navigate as more declarations are added.

Easier said than done, but yes, it is something to aim for.

I learned that GitHub Pages is case-sensitive unlike my Mac, so image file names in HTML must match exactly.

Correct.

Be very careful with the next few assignments and make sure that you check and double check all the common mistakes, because they are double the deduction now!

Yes, they are.

I looked ahead and saw that the next assignment is to create light/dark mode styling, so I’m currently thinking about how I can maintain the theme with color changes.

I look forward to seeing it.
