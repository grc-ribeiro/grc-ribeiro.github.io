I wanted to see if there was a way to avoid having to write input:nth-of-type(#):checked ~ main figure:nth-of-type(#) for each figure, and I found out that there isn’t a way to loop or use dynamic matching in CSS. However, you can write a loop if you style using SCSS and use a tool to help compile it to CSS after.

There are many ways you can automate the writing of the selector, but it’s a shame that you can’t write a simplified selection to automate the rendering. Thanks for investigating this, Tra!

I haven’t used the main tag too much, so I’m curious how it generally works. Is it like a div or section tag?

From MDN: “The <main> HTML element represents the dominant content of the <body> of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.” MDN and W3Schools are excelent references, but you can also look it up stuff. Keep in mind that HTML is not a secret language that we only use in this department. There’s plenty of resources for you to expand your knowledge when you find something that sparks your curiosity.
