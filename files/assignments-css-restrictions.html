Website redesign: CSS restrictions

  1. You should keep your CSS to under 300 lines. (400 at the very most, but 300 should be more than enough.)

  2. Don’t cheat. Keep writing your CSS as we have been:
     
    element‑or‑class {        one line for declaring the element or class
          property: value;
          property: value;        keep one property per line
          property: value;
       }                                               closing bracket in its own line
                                                           skip one line before the next element or class

Tips for keeping your CSS economical

  1. Reuse your classes as much as possible. Fewer classes help with consistency anyway.

  2. Only declare what you need to. (You don’t need to declare white background or black font.)

  3. Give properties to all elements, using the body:
     
    body {
          color: rgb(40, 40, 40);
          font‑family: serif;
       }

  4. Group elements that share same properties. If you have many element, with commas, you will be declaring the values for all of them:
     
    h1, h2, h3, h4 {
          color: rgb(180, 0, 0);
          font‑family: sans‑serif;
       }
