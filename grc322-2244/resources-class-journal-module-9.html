GRC 322: Resources

Class journal, module 9

I figured I would look at the example type specimens for inspiration but found that they are all so different. It’s difficult to know which direction to go with mine, so I’m thinking of doing some sketches like storyboarding so I can visualize content organization and hierarchy.

That’s a good idea! Sketching never worked well for me, but I know several designers that always start with sketching. Make sure you have your references before you start sketching. Then you know where you’re going.

For the type comparison, I thought red and yellow would be cool to use for the color palette but it just ended up reminding everyone of Mcdonald’s. Avoid red and yellow when designing non-fast food related stuff.

Two groups ran into this problem.

I find that I lean towards the same design aesthetic often and want to push myself out of my comfort zone for this project.

Make sure you have references that are different from what you normally do.

I like playing with different colors because sometimes taking a risk ends up working. I hope that doesn’t count as being too original because in my opinion it still looks good.

Nothing wrong with being original as long as it’s good. If you aim at being good, there’s a good chance you’ll be original too. If you aim at being original rather than good, you’ll most likely be neither.

Are there any design choices we should avoid using other than ampersands?

There are way worse things to do than using an ampersand. Avoid centering or aligning type to right. Sometimes it’s harder to design with left‑alignment, but it’s worth it. Another possibility is breaking all possible alignment. Oh, and avoid small caps with uppercase; they look awful.
